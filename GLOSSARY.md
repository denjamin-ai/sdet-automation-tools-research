# Глоссарий — Инструменты автоматизации в тестировании

> Автоматически обновляемый словарь технических терминов.  
> Формат: **Термин** — определение.

---

## A

**Allure Report** — открытый фреймворк для генерации наглядных отчётов о результатах тестирования. Поддерживает интеграцию с JUnit, TestNG, Pytest и другими фреймворками.

**API (Application Programming Interface)** — программный интерфейс приложения; набор правил и протоколов для взаимодействия между программными компонентами.

**Automation QA** — специалист по автоматизированному тестированию, ответственный за написание и поддержку автоматических тестов.

## B

**BDD (Behavior-Driven Development)** — методология разработки через поведение, где тестовые сценарии описываются на естественном языке (Gherkin).

## C

**CI/CD (Continuous Integration / Continuous Delivery)** — практики непрерывной интеграции и непрерывной доставки кода, обеспечивающие автоматическую сборку, тестирование и развёртывание.

**CheckStyle** — инструмент статического анализа Java-кода для проверки соответствия стандартам кодирования.

## D

**DevOps** — методология объединения процессов разработки (Development) и эксплуатации (Operations) для ускорения поставки ПО.

## F

**Framework (Фреймворк)** — программная платформа, предоставляющая структуру и набор инструментов для разработки приложений или автотестов.

## G

**GitLab CI/CD** — встроенная система непрерывной интеграции и доставки в платформе GitLab, управляемая через файл `.gitlab-ci.yml`.

**Grafana** — открытая платформа для визуализации метрик и мониторинга, поддерживающая множество источников данных (InfluxDB, Prometheus и др.).

## I

**IDE (Integrated Development Environment)** — интегрированная среда разработки (например, IntelliJ IDEA, VS Code).

## L

**Legacy (Легаси)** — устаревший код или проект, который продолжает использоваться, но требует модернизации.

## M

**MCP (Model Context Protocol)** — протокол взаимодействия с языковыми моделями, позволяющий интегрировать AI в рабочие процессы.

**Mermaid** — язык разметки для описания диаграмм и блок-схем в текстовом формате, рендерится в Markdown.

## O

**Onboarding (Онбординг)** — процесс введения нового сотрудника в проект, включающий обучение инструментам, процессам и кодовой базе.

## P

**Pipeline (Пайплайн)** — автоматизированная последовательность этапов обработки: сборка, тестирование, развёртывание.

**PMD** — инструмент статического анализа кода, выявляющий потенциальные проблемы: неиспользуемые переменные, пустые блоки catch, дублирование кода.

**Plugin (Плагин)** — программный модуль, расширяющий функциональность основного приложения без изменения его исходного кода.

## R

**RAG (Retrieval-Augmented Generation)** — архитектура AI, сочетающая поиск релевантных документов с генерацией ответа языковой моделью.

**Regression Testing (Регрессионное тестирование)** — повторное тестирование ранее проверенной функциональности после внесения изменений.

## S

**SDET (Software Development Engineer in Test)** — инженер по разработке ПО в тестировании; специалист, сочетающий навыки разработки и тестирования для создания инфраструктуры, инструментов и автоматизации процессов QA.

**Smoke Testing (Смок-тестирование)** — поверхностная проверка основной функциональности после сборки или развёртывания.

**Stand (Стенд)** — изолированная среда для тестирования или разработки (dev, staging, QA).

## T

**TMS (Test Management System)** — система управления тестированием (например, TestRail, Allure TestOps, Xray), хранящая тест-кейсы, тест-планы и результаты прогонов.

## W

**Webhook** — механизм уведомления внешних систем о событиях через HTTP-запросы в реальном времени.
